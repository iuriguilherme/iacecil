# Research Pitfalls - Known Patterns to Avoid

## Known Pitfalls

### Pitfall 1: Configuration Scope Assumptions
**What**: Assuming global configuration means no project-scoping exists.
**Prevention**: Verify ALL scopes (User, Project, Local, Workspace, Environment).

### Pitfall 2: "Search for X" Vagueness
**What**: Vague research instructions without specifying authoritative sources.
**Prevention**: Provide exact URLs to official documentation in `<sources>`.

### Pitfall 3: Deprecated vs Current Features
**What**: Finding old documentation and concluding a feature doesn't exist.
**Prevention**: Review changelogs, check publication dates, and cross-reference multiple sources.

### Pitfall 4: Tool-Specific Variations
**What**: Conflating capabilities across different tools or environments.
**Prevention**: Document which specific environment supports which features.

### Pitfall 5: Confident Negative Claims Without Citations
**What**: Stating something isn't possible without official source verification.
**Prevention**: Require explicit evidence of absence before making negative claims.

### Pitfall 6: Missing Enumeration
**What**: Investigating open-ended scope without listing possibilities first.
**Prevention**: Enumerate ALL known options before starting investigation.

### Pitfall 7: Single-Source Verification
**What**: Relying on a single source for critical claims.
**Prevention**: Require multiple authoritative sources for verification.

### Pitfall 8: Assumed Completeness
**What**: Assuming search results are complete and authoritative without checking dates.
**Prevention**: Verify publication dates and source authority for every finding.

## Red Flags in Research Outputs
🚩 Zero "Not Found" results (unrealistic)
🚩 No confidence indicators
🚩 Missing URLs for citations
🚩 Definitive statements without evidence
🚩 Incomplete enumeration of checklist items
